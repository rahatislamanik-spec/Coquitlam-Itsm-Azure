Service Phase Purpose Design Decision
Azure Static Web AppsPhase 1Student portalCDN-backed — fast for international students regardless of timezone
Azure Functions (HTTP)Phase 1Request intakeServerless — pay per execution, scales during registration peaks
Key Vault + Managed IdentityAll phasesSecret managementZero hardcoded credentials — enterprise security standard
Azure Event GridPhase 2Event routingDecouples intake from processing — loose coupling, independent failure
Azure Service BusPhase 2Queue processingDead-letter queue — no ticket silently lost. Chosen over Storage Queue
Azure Functions (Queue)Phase 2Backend processorTriggered only when work exists — true serverless efficiency
Azure Cosmos DBPhase 3Ticket recordsRich queries + serverless billing. Chosen over Table Storage
Azure Blob StoragePhase 3File attachmentsCheap, durable, globally replicated — full audit trail forever
Azure Communication SvcPhase 4Email confirmationsInstant automated acknowledgement — no student left in silence
Azure Logic AppsPhase 4SLA escalationAuto-escalates breached tickets — zero manual monitoring required
Power AutomatePhase 4Teams notificationsMeets staff where they already work — native M365 integration
Application InsightsPhase 5Performance trackingCustom events: TicketSubmitted, SLABreached, EscalationTriggered
Log Analytics WorkspacePhase 5Log aggregationKQL queries surface weekly patterns and SLA compliance reports
Azure MonitorPhase 5Alert rulesSelf-monitoring — detects failures before any human notices
